How to Use This Converter

We designed this tool with simplicity in mind. Here is your guide to creating PDFs in seconds:

Option 1: Converting a Word File

  1. Select the "Upload File" Tab: This is selected by default.
  2. Drag & Drop: Drag your .docx file into the dotted box, or click to browse your folders.
  3. Choose Paper Size: Use the dropdown menu to select A4 (Standard), A3 (Large), or Letter (US Standard).
  4. Convert & Download: Click "Convert to Preview" to check the content, then hit "Download PDF" to save it.

Option 2: Converting Pasted Text

  1. Select the "Paste Text" Tab: Click the button at the top of the tool box.
  2. Type or Paste: Enter your content into the text box. This is perfect for quick notes, emails, or content copied from websites.
  3. Convert: Click "Convert to Preview". The tool will automatically format your text into paragraphs.
  4. Download: Select your paper size and download the resulting PDF.

New Feature: Custom Paper Sizes (A4, A3, Letter)

One size does not fit all. Standard converters often lock you into A4 size, which can be frustrating for users in North America (who use Letter) or designers needing larger formats (A3). Our tool allows dynamic resizing:

  • A4 (210 x 297 mm): The global standard for business documents, letters, and forms.
  • Letter (8.5 x 11 in): The standard size in the United States, Canada, and Mexico.
  • A3 (297 x 420 mm): Double the size of A4. Ideal for large tables, diagrams, or posters.

When you select a size, the PDF generator re-calculates the layout engine (using html2pdf.js) to ensure your text flows correctly within the margins of the chosen format.

Why Client-Side Processing is Safer

In an era of data breaches, uploading files to unknown servers is risky. When you use server-side converters, your file travels across the internet to a remote computer. SecurePDF.js is different. It is a client-side tool.

This means the conversion logic (the JavaScript code) runs inside your browser (Chrome, Edge, Firefox, etc.). Your document never leaves your computer. You could disconnect your internet after loading the page, and the tool would still function perfectly. This is critical for legal documents, medical records, or personal diaries.

Deep Dive: The Technology Stack

This single-file tool is a marvel of modern web development, utilizing two primary libraries:

Mammoth.js (for DOCX)

We use Mammoth.js to parse the complex XML structure of Microsoft Word files. Unlike other parsers that try to guess the layout, Mammoth focuses on semantics. It ensures that a "Heading 1" in Word becomes an <h1> in HTML, preserving the document structure rather than just visual positioning.

Html2Pdf.js (for PDF Generation)

This powerful library takes the HTML previewâ€”whether generated from a Word doc or your pasted textâ€”and renders it onto a PDF canvas. It handles the pagination (breaking text across pages) and respects the paper size dimensions (A4/A3/Letter) that you select in the UI.

Frequently Asked Questions (FAQ)

Can I paste rich text (images/bold) directly?

The "Paste Text" option is currently optimized for plain text. It will preserve paragraphs and line breaks. To convert images and complex formatting, we recommend saving your content in a Word document first and using the "Upload File" tab.

Does changing the paper size affect the font size?

No. The font size remains consistent (usually 12pt equivalent). Changing the paper size changes the amount of space available on the page, meaning an A3 page will hold significantly more text per page than an A4 page.

Is this tool compatible with mobile devices?

Absolutely. Because it uses standard web technologies, you can use this on an Android phone, iPhone, or iPad. The "Paste Text" feature is particularly useful on mobile for saving notes as PDFs.

Why is there no "Upload" button?

There is no upload button because there is no server to upload to! You are simply opening the file into the browser's memory. This is a security feature, not a bug.

Hosting on GitHub Pages

This tool is delivered as a single HTML file. This architecture is perfect for GitHub Pages. Developers can fork the repository, audit the code to ensure security, and host their own version completely free of charge. It eliminates the need for backend servers (like Node.js or PHP) or databases, making the tool incredibly fast and reliable.

Troubleshooting

  • Text cut off in PDF: Try changing the paper size to a larger format (e.g., A4 to A3) or reducing the margins in your source document.
  • Special Characters: The tool uses UTF-8 encoding, so it supports most languages. If you see strange symbols, ensure your original file uses standard fonts like Arial or Times New Roman.
  • Large Files: Processing very large Word documents (50+ pages) might slow down the browser momentarily. Be patient; the browser is doing heavy lifting!
