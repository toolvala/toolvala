2. How to Use This Tool

We have optimized the interface for two distinct types of user intent: simple corrections and complex data transformations.

Mode 1: Single Replace (Standard Use)

Use this tab for quick, one-off corrections. Simply enter the target word and the replacement. Our engine supports "Whole Word" logic, ensuring that if you replace "Cat," the tool won't accidentally change "Category" to "Dogegory."

Mode 2: Bulk Swap (Find -> Replace)

This is the "Power User" mode. It allows you to define a list of rules to be executed sequentially. For example, if you have a list of abbreviations (Jan, Feb, Mar) and you want them expanded to full month names, you can enter them as Jan -> January and so on. The tool will iterate through the entire document and perform all swaps in a single pass.

5. Frequently Asked Questions (FAQ)

1. What is the difference between Match Case and Whole Words? ▼
'Match Case' ensures 'cat' and 'Cat' are treated differently. 'Whole Words' ensures the tool only replaces standalone words, not characters found inside other words (like 'cat' inside 'catalog').
2. Can I use Regex to find and remove text? ▼
Yes! Simply enter your Regex pattern in the Find box and leave the Replace box empty. Clicking execute will strip all matching patterns from your document.
3. How does the Bulk Mode syntax work? ▼
Use the format: [Old Text] -> [New Text]. Each rule should be on a new line. The tool will process them from top to bottom.
4. Is there a limit to the text size I can process? ▼
The limit is determined by your browser's RAM. Most modern browsers can handle several megabytes of text (millions of words) without any lag.
5. Does this tool support Devanagari or other scripts? ▼
Yes. Since the tool uses standard UTF-8 encoding, you can find and replace Hindi, Marathi, Arabic, or any other non-Latin characters perfectly.
6. Is it safe for confidential source code? ▼
Absolutely. Your code is processed in the browser window's temporary memory and never uploaded to the internet. It is 100% private.
